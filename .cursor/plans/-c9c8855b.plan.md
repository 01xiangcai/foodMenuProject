# 修正菜品图片显示逻辑

## 一、显示优先级规则

```
local_image 有值 → 显示 local_image
    ↓ 否
image 有值 → 显示 image
    ↓ 否
显示默认图片
```

**位置**: [`food_menu_uniapp/static/`](food_menu_uniapp/static/)

- 建议尺寸: 400x400 或 600x600

### 2.2 创建图片处理工具函数

**新建文件**: [`food_menu_uniapp/src/utils/image.js`](food_menu_uniapp/src/utils/image.js)

```javascript
/**
 * 获取菜品图片URL
 * @param {Object} dish - 菜品对象
 * @param {string} dish.localImage - 本地图片路径
 * @param {string} dish.image - OSS图片路径
 * @returns {string} 图片URL
 */
export function getDishImageUrl(dish) {
  if (dish.localImage) {
    return dish.localImage;
  }
  if (dish.image) {
    return dish.image;
  }
  return '/static/default-dish.png';
}

/**
 * 获取菜品图片URL（兼容不同字段名）
 * @param {Object} dish - 菜品对象
 * @param {string} dish.local_image - 本地图片路径（下划线命名）
 * @param {string} dish.localImage - 本地图片路径（驼峰命名）
 * @returns {string} 图片URL
 */
export function getDishImage(dish) {
  const localImage = dish.localImage || dish.local_image;
  if (localImage) {
    return localImage;
  }
  if (dish.image) {
    return dish.image;
  }
  return '/static/default-dish.png';
}
```

## 四、后端接口优化（可选）

### 4.1 统一返回字段

**文件**: [`food_menu/src/main/java/com/yao/food_menu/controller/DishController.java`](food_menu/src/main/java/com/yao/food_menu/controller/DishController.java)

**建议**: 在返回数据时，后端可以添加一个计算字段 `displayImage`，统一处理显示逻辑

```java
public DishVO convertToVO(Dish dish) {
    DishVO vo = new DishVO();
    // ... 其他字段映射
    vo.setLocalImage(dish.getLocalImage());
    vo.setImage(dish.getImage());
    
    // 添加显示用的图片字段
    String displayImage = dish.getLocalImage();
    if (StringUtils.isBlank(displayImage)) {
        displayImage = dish.getImage();
    }
    vo.setDisplayImage(displayImage);
    
    return vo;
}
```

这样前端可以直接使用 `dish.displayImage`，但这是可选的优化。

## 七、注意事项

1. **字段命名**: 注意后端可能返回 `localImage`（驼峰）或 `local_image`（下划线），工具函数需要兼容
2. **默认图片路径**: 

                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                - 小程序使用相对路径 `/static/default-dish.png`
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                - 管理后台使用 `/default-dish.png` 或 `@/assets/default-dish.png`

3. **图片加载失败**: 考虑使用 `@error` 事件处理图片加载失败的情况
4. **性能**: 默认图片应该尽量小（< 50KB），避免影响加载速度
5. **一致性**: 所有显示菜品图片的地方都应该使用相同的逻辑

## 八、后续与多图功能的关系

完成这个修正后，多图功能实施时：

- `local_image` 继续作为主图字段
- 新增 `local_images` 字段存储所有图片
- 图片显示逻辑保持不变，只是数据来源会更丰富
- 工具函数可以扩展支持多图场景
