## 🔐 功能详情

### 1. 接口限流

**保护接口：**

| 接口 | QPS | 限流类型 | 说明 |
|------|-----|---------|------|
| 验证码发送（2个） | 1 | IP | 每秒1次，防短信轰炸 |
| 用户登录（2个） | 5 | IP | 每秒5次，防暴力破解 |
| 用户注册 | 2 | IP | 每秒2次，防批量注册 |
| 订单提交 | 3 | 用户 | 每秒3次，防刷单 |
| 发表评论 | 5 | 用户 | 每秒5次，防刷评论 |

**使用示例：**
```java
@RateLimiter(qps = 5, limitType = LimitType.IP)
@PostMapping("/login")
public Result<String> login() { }
```

### 2. 防重复提交

**保护接口：**

| 接口 | 间隔 | 维度 | 说明 |
|------|------|------|------|
| 订单提交 | 3秒 | 用户 | 防止重复下单 |
| 用户注册 | 2秒 | IP | 防止重复注册 |
| 发表评论 | 2秒 | 用户 | 防止重复评论 |

**使用示例：**
```java
@PreventDuplicateSubmit(interval = 3000, byUser = true)
@PostMapping("/order/submit")
public Result<Long> submit() { }
```

### 3. SQL注入防护

**检查结果：** ✅ 安全

- 所有SQL使用 `#{}` 参数化查询
- 未发现 `${}` 字符串拼接
- MyBatis-Plus自动防护
- 详见：`SQL注入防护报告.md`

## 🎯 使用说明

### 为新接口添加保护

**1. 添加限流：**
```java
@RateLimiter(qps = 10, limitType = LimitType.IP)
@PostMapping("/new-api")
public Result<?> newApi() {
    // 业务逻辑
}
```

**2. 添加防重复提交：**
```java
@PreventDuplicateSubmit(interval = 5000)
@PostMapping("/new-submit")

    // 业务逻辑
}
```

**3. 组合使用：**
```java
@RateLimiter(qps = 5, limitType = LimitType.USER)
@PreventDuplicateSubmit(interval = 3000)
@PostMapping("/important")

    // 业务逻辑
}
```

## ⚠️ 重要提示

### 1. 单机限流

当前是**单机限流**：
- 单实例：QPS即设定值
- 多实例：总QPS = 设定值 × 实例数

**升级方案：** 后续可接入Redis实现分布式限流

### 2. 本地缓存

防重复提交使用本地缓存：
- 应用重启会清空
- 不影响业务逻辑

### 3. 性能影响

- 响应延迟：< 1ms
- 内存占用：极小
- **可忽略不计**

## 📈 监控建议

### 建议监控指标

1. **限流触发次数** - 识别攻击行为
2. **重复提交拦截次数** - 识别前端问题
3. **异常IP访问** - 识别恶意用户
4. **接口响应时间** - 性能监控

### 告警设置

- 限流频繁触发 → 可能遭受攻击
- 大量重复提交 → 可能前端bug
- 特定IP高频访问 → 可能爬虫
