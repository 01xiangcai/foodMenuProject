# 敏感信息管理说明

## 📋 概述

本项目已实施敏感信息加密管理方案，使用 Jasypt 对配置文件中的敏感信息进行加密，避免明文存储数据库密码、API密钥等敏感数据。

---

## 🔐 已加密的敏感信息

1. **JWT密钥** (`jwt.secret`)
2. **数据库密码** (`spring.datasource.password`)
3. **阿里云OSS访问密钥** (`aliyun.oss.access-key-id`)
4. **阿里云OSS访问密钥密文** (`aliyun.oss.access-key-secret`)

---

## 🛠️ 使用方法

### 1. 生成加密值

运行测试类生成加密值：

```bash
cd food_menu
mvn test -Dtest=JasyptEncryptorTest
```

或在IDE中直接运行 `JasyptEncryptorTest.java`

**输出示例：**
```
明文: root
密文: xMjQwMDAwMDAw...
配置文件格式: ENC(xMjQwMDAwMDAw...)
```

### 2. 更新配置文件

将生成的加密值替换到配置文件中：

**application-prod.yml示例：**
```yaml
spring:
  datasource:
    password: ENC(xMjQwMDAwMDAw...)  # 替换为实际加密值

jwt:
  secret: ENC(加密后的JWT密钥)

aliyun:
  oss:
    access-key-id: ENC(加密后的AccessKeyId)
    access-key-secret: ENC(加密后的AccessKeySecret)
```

### 3. 设置解密密钥

生产环境启动时，必须通过环境变量或启动参数传入加密密钥。

#### 方法一：环境变量（推荐）

**Linux/Mac:**
```bash
export JASYPT_ENCRYPTOR_PASSWORD=FoodMenuSecretKey2024!@#
java -jar food_menu-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

**Windows:**
```cmd
set JASYPT_ENCRYPTOR_PASSWORD=FoodMenuSecretKey2024!@#
java -jar food_menu-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

**Docker:**
```bash
docker run -e JASYPT_ENCRYPTOR_PASSWORD=FoodMenuSecretKey2024!@# \
  -p 8080:8080 food-menu:latest
```

#### 方法二：启动参数

```bash
java -jar -Djasypt.encryptor.password=FoodMenuSecretKey2024!@# \
  food_menu-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

#### 方法三：Systemd服务配置

编辑 `/etc/systemd/system/food-menu.service`：

```ini
[Service]
Environment="JASYPT_ENCRYPTOR_PASSWORD=FoodMenuSecretKey2024!@#"
ExecStart=/usr/bin/java -jar /opt/food-menu/food_menu-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

```yaml
  secret: ENC(加密值)

  oss:
    access-key-id: ENC(加密值)
    access-key-secret: ENC(加密值)
```

## 🔑 密钥管理最佳实践

### 1. 加密密钥要求

- **长度：** 建议16位以上
- **复杂度：** 包含大小写字母、数字、特殊字符
- **示例：** `FoodMenuSecretKey2024!@#`

### 2. 密钥存储

**禁止：**
- ❌ 硬编码在代码中
- ❌ 提交到Git仓库
- ❌ 明文记录在文档中

**推荐：**
- ✅ 使用环境变量
- ✅ 使用密钥管理服务（AWS Secrets Manager、阿里云KMS）
- ✅ 使用配置中心（Nacos、Apollo）
- ✅ 团队内部密钥保管系统

### 3. 密钥轮换

定期更换加密密钥（建议3-6个月）：

1. 生成新的加密密钥
2. 使用新密钥重新加密所有敏感信息
3. 更新配置文件
4. 更新生产环境的环境变量
5. 重启服务

| 环境变量名 | 说明 | 示例值 | 保管人 |
|-----------|------|--------|-------|
| `JASYPT_ENCRYPTOR_PASSWORD` | Jasypt加密密钥 | FoodMenuSecretKey2024!@# | 运维负责人 |

| OSS AccessKeySecret | 阿里云OSS访问密钥 | （密钥） | 运维负责人 |

## 🚨 安全注意事项

### 1. Git安全

确保以下文件已添加到 `.gitignore`：

```gitignore
# 敏感配置文件
application-prod.yml
*.env
.env.*
secrets/

# 本地密钥文件
jasypt-keys.txt
```

### 2. 日志安全

确保敏感信息不会被记录到日志：

```java
// ✅ 正确：不记录敏感信息
log.info("用户登录成功, userId: {}", userId);

// ❌ 错误：记录了密码
log.info("用户登录, username: {}, password: {}", username, password);
```

### 3. 异常处理

不要在异常信息中暴露敏感信息：

```java
try {
    // 数据库操作
} catch (Exception e) {
    // ✅ 正确：记录异常类型和消息
    log.error("数据库操作失败: {}", e.getMessage());
    
    // ❌ 错误：可能暴露连接字符串（包含密码）
    log.error("连接失败: {}", dataSource.getUrl());
}
```

## 🔧 故障排查

### 问题1：启动失败，提示解密错误

**原因：** 未设置 `JASYPT_ENCRYPTOR_PASSWORD` 或密钥不正确

**解决：**
```bash
export JASYPT_ENCRYPTOR_PASSWORD=你的加密密钥
```

### 问题2：数据库连接失败

**原因：** 数据库密码加密值不正确

**解决：**
1. 使用 `JasyptEncryptorTest` 重新生成加密值
2. 更新配置文件
3. 重启应用

### 问题3：JWT Token验证失败

**原因：** JWT密钥配置不正确

**解决：**
1. 检查 `jwt.secret` 配置
2. 确保加密值正确
3. 重新生成Token
