# 日志系统使用说明

## 概述

本项目已集成完整的日志系统，包括：
- ✅ Logback日志框架配置
- ✅ 按日期和大小滚动日志文件
- ✅ 区分开发和生产环境的日志级别
- ✅ 敏感信息自动脱敏
- ✅ 分类日志文件（all.log、info.log、warn.log、error.log）

## 日志文件位置

### 开发环境
- 日志路径: `logs/food_menu/`
- 输出目标: 控制台 + 文件

### 生产环境
- 日志路径: `/var/log/food_menu/`
- 输出目标: 仅文件（不输出到控制台）

## 日志文件说明

| 文件名 | 说明 | 保留时间 | 最大大小 |
|--------|------|----------|----------|
| all.log | 所有级别日志 | 30天 | 100MB/文件 |
| info.log | INFO级别日志 | 30天 | 100MB/文件 |
| warn.log | WARN级别日志 | 30天 | 100MB/文件 |
| error.log | ERROR级别日志 | 60天 | 100MB/文件 |

## 日志级别配置

### 开发环境 (默认/dev)
```yaml
logging:
  level:
    root: INFO
    com.yao.food_menu: DEBUG          # 应用日志DEBUG级别
    com.yao.food_menu.mapper: DEBUG   # 显示SQL日志
    org.springframework: INFO
    org.apache.ibatis: DEBUG
```

### 生产环境 (prod)
```yaml
logging:
  level:
    root: INFO
    com.yao.food_menu: INFO           # 应用日志INFO级别
    com.yao.food_menu.mapper: WARN    # 不显示SQL日志
    org.springframework: WARN
    org.apache.ibatis: WARN
```

## 日志使用示例

### 1. 在类中引入日志

使用Lombok的 `@Slf4j` 注解：

```java
import lombok.extern.slf4j.Slf4j;

@Service
@Slf4j
public class UserServiceImpl {
    // 自动生成 log 对象
}
```

或手动创建：

```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class UserServiceImpl {
    private static final Logger log = LoggerFactory.getLogger(UserServiceImpl.class);
}
```

### 2. 日志级别使用场景

```java
// DEBUG - 调试信息，仅开发环境输出
log.debug("用户查询参数: {}", queryDto);

// INFO - 业务流程信息
log.info("用户登录成功: userId={}, username={}", userId, username);

// WARN - 警告信息，不影响系统运行
log.warn("用户登录失败-密码错误: username={}", username);

// ERROR - 错误信息，需要关注和处理
log.error("数据库操作异常: {}", e.getMessage(), e);
```

### 3. 敏感信息处理

#### 自动脱敏
日志系统会自动脱敏以下信息：

```java
// 手机号：13812345678 -> 138****5678
log.info("发送验证码到: {}", phone);

// 密码：password=123456 -> password=******
log.info("用户信息: {}", user);  // User对象中的password字段

// Token：token=eyJhbG... -> token=eyJh****
log.info("请求头: {}", headers);

// 身份证号：123456199001011234 -> 123456********1234
log.info("身份证: {}", idCard);

// 邮箱：example@test.com -> exa***@test.com
log.info("邮箱: {}", email);
```

#### 手动脱敏
使用 `SensitiveDataConverter` 工具类：

```java
import com.yao.food_menu.common.util.SensitiveDataConverter;

// 脱敏手机号
String maskedPhone = SensitiveDataConverter.maskPhone("13812345678");
// 结果: 138****5678

// 脱敏邮箱
String maskedEmail = SensitiveDataConverter.maskEmail("example@test.com");
// 结果: exa***@test.com

// 脱敏身份证
String maskedIdCard = SensitiveDataConverter.maskIdCard("123456199001011234");
// 结果: 123456********1234

// 脱敏银行卡
String maskedCard = SensitiveDataConverter.maskBankCard("1234567890123456");
// 结果: 1234********3456
```

### 4. 异常日志记录

```java
try {
    // 业务代码
} catch (Exception e) {
    // 记录异常堆栈信息（第三个参数传入异常对象）
    log.error("用户登录异常: userId={}, 错误信息={}", userId, e.getMessage(), e);
}
```

### 5. 使用 @SensitiveData 注解

在实体类中标记敏感字段：

```java
import com.yao.food_menu.common.annotation.SensitiveData;
import com.yao.food_menu.common.annotation.SensitiveData.SensitiveType;

public class User {
    @SensitiveData(type = SensitiveType.PHONE)
    private String phone;
    
    @SensitiveData(type = SensitiveType.PASSWORD)
    private String password;
    
    @SensitiveData(type = SensitiveType.ID_CARD)
    private String idCard;
    
    @SensitiveData(type = SensitiveType.EMAIL)
    private String email;
}
```

## 最佳实践

### 1. Controller层日志

```java
@RestController
@Slf4j
public class UserController {
    
    @PostMapping("/login")
    public Result<String> login(@RequestBody LoginDto loginDto) {
        // 记录请求（不输出密码等敏感信息）
        log.info("用户登录请求: username={}", loginDto.getUsername());
        
        try {
            String token = userService.login(loginDto);
            log.info("用户登录成功: username={}", loginDto.getUsername());
            return Result.success(token);
        } catch (Exception e) {
            log.warn("用户登录失败: username={}, 原因={}", loginDto.getUsername(), e.getMessage());
            return Result.error(e.getMessage());
        }
    }
}
```

### 2. Service层日志

```java
@Service
@Slf4j
public class UserServiceImpl implements UserService {
    
    @Override
    public String login(LoginDto loginDto) {
        log.debug("开始处理登录: username={}", loginDto.getUsername());
        
        User user = userMapper.selectByUsername(loginDto.getUsername());
        if (user == null) {
            log.warn("用户不存在: username={}", loginDto.getUsername());
            throw new RuntimeException("用户不存在");
        }
        
        log.info("用户登录成功: userId={}, username={}", user.getId(), user.getUsername());
        return jwtUtil.generateToken(user.getId());
    }
}
```

### 3. 异常处理器日志

```java
@ControllerAdvice
@Slf4j
public class GlobalExceptionHandler {
    
    @ExceptionHandler(Exception.class)
    public Result<String> handleException(Exception e) {
        // 记录完整的异常堆栈信息
        log.error("全局异常捕获: {}", e.getMessage(), e);
        return Result.error("系统异常，请稍后重试");
    }
}
```

```bash

```

### 日志分析

```bash

```

## 注意事项

1. **敏感信息保护**
   - 永远不要在日志中输出完整的密码、Token等敏感信息
   - 使用脱敏工具处理手机号、身份证等信息
   - 生产环境关闭DEBUG级别日志

2. **日志级别选择**
   - DEBUG: 仅用于开发调试
   - INFO: 记录关键业务流程
   - WARN: 记录可能的问题
   - ERROR: 记录需要处理的错误

3. **性能考虑**
   - 避免在循环中使用DEBUG日志
   - 大量数据不要完整输出到日志
   - 生产环境使用INFO及以上级别

4. **日志维护**
   - 定期检查日志文件大小
   - 关注错误日志，及时处理问题
   - 清理过期的历史日志（自动清理）

## 故障排查

### 控制台中文乱码

**Windows系统**：
- 已配置控制台输出使用GBK编码，避免乱码
- 如果仍有乱码，可在IDEA中设置：
  1. `Help` → `Edit Custom VM Options`
  2. 添加：`-Dfile.encoding=UTF-8`
  3. 重启IDEA

**Linux系统**：
- 如果需要UTF-8编码，修改 `logback-spring.xml` 中CONSOLE的charset为UTF-8
- 或者使用环境变量：`export JAVA_TOOL_OPTIONS=-Dfile.encoding=UTF-8`

**PowerShell乱码**：
```powershell
# 临时修改PowerShell编码为UTF-8
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
chcp 65001

# 或者使用GBK（推荐）
chcp 936
```

### 日志文件未生成

1. 检查目录权限
```bash
# Linux
mkdir -p /var/log/food_menu
chmod 755 /var/log/food_menu

# Windows
mkdir logs\food_menu
```

2. 检查配置文件
- 确认 `logback-spring.xml` 在 `resources` 目录下
- 确认 `application.yml` 中的日志配置正确

### SQL日志未显示

生产环境默认关闭SQL日志，修改配置：

```yaml
logging:
  level:
    com.yao.food_menu.mapper: DEBUG
```

或使用MyBatis-Plus的日志：

```yaml
mybatis-plus:
  configuration:
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```
