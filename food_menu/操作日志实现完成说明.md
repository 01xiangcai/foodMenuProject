## 🚀 部署步骤

### 步骤1：创建数据库表

执行以下SQL创建 `operation_log` 表：

```sql
-- 切换到数据库
USE food_menu;

-- 创建操作日志表
CREATE TABLE `operation_log` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键',
  `operation_type` varchar(50) DEFAULT NULL COMMENT '操作类型：INSERT/UPDATE/DELETE/QUERY/LOGIN/LOGOUT/EXPORT/IMPORT',
  `operation_module` varchar(50) DEFAULT NULL COMMENT '操作模块：订单/菜品/用户/分类/轮播图等',
  `operation_desc` varchar(500) DEFAULT NULL COMMENT '操作描述',
  `method_name` varchar(200) DEFAULT NULL COMMENT '方法名',
  `request_method` varchar(10) DEFAULT NULL COMMENT '请求方式：GET/POST/PUT/DELETE',
  `request_url` varchar(500) DEFAULT NULL COMMENT '请求URL',
  `request_params` text COMMENT '请求参数（JSON格式）',
  `response_result` text COMMENT '响应结果（JSON格式）',
  `operator_id` bigint DEFAULT NULL COMMENT '操作人ID',
  `operator_name` varchar(100) DEFAULT NULL COMMENT '操作人姓名',
  `operator_type` varchar(20) DEFAULT NULL COMMENT '操作人类型：ADMIN(管理员)/USER(普通用户)/SYSTEM(系统)',
  `ip_address` varchar(50) DEFAULT NULL COMMENT 'IP地址',
  `user_agent` varchar(500) DEFAULT NULL COMMENT '用户代理',
  `execution_time` bigint DEFAULT NULL COMMENT '执行时长(毫秒)',
  `status` int DEFAULT 1 COMMENT '状态：0失败 1成功',
  `error_msg` text COMMENT '错误信息',
  `family_id` bigint DEFAULT NULL COMMENT '家庭ID',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_operator_id` (`operator_id`),
  KEY `idx_operation_type` (`operation_type`),
  KEY `idx_operation_module` (`operation_module`),
  KEY `idx_create_time` (`create_time`),
  KEY `idx_family_id` (`family_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='操作日志表';
```

或者直接执行SQL文件：

```bash
mysql -u root -p food_menu < food_menu/src/main/resources/sql/operation_log.sql
```

## 📝 如何添加更多日志

在需要记录日志的Controller方法上添加 `@OperationLog` 注解：

```java
@OperationLog(
    operationType = OperationLog.OperationType.INSERT,
    operationModule = "分类",
    operationDesc = "添加分类",
    recordParams = true,
    recordResult = false
)
@PostMapping
public Result<String> save(@RequestBody Category category) {
    categoryService.save(category);
    return Result.success("添加成功");
}
```

## 🔍 查询操作日志

### API接口

```bash
# 1. 分页查询
GET /admin/operationLog/page
参数：
  - page: 页码
  - pageSize: 每页大小
  - operationType: 操作类型
  - operationModule: 操作模块
  - operatorId: 操作人ID
  - operatorName: 操作人姓名
  - status: 状态
  - startTime: 开始时间
  - endTime: 结束时间

# 2. 查询详情
GET /admin/operationLog/{id}

# 3. 删除日志（慎用）
DELETE /admin/operationLog/{id}

# 4. 批量删除
DELETE /admin/operationLog/batch?endTime=2024-01-01 00:00:00
```

### 数据库查询

```sql
-- 查询最近10条操作日志
SELECT * FROM operation_log ORDER BY create_time DESC LIMIT 10;

-- 查询某个用户的所有操作
SELECT * FROM operation_log WHERE operator_id = 1 ORDER BY create_time DESC;

-- 查询失败的操作
SELECT * FROM operation_log WHERE status = 0 ORDER BY create_time DESC;

-- 统计各模块操作次数
SELECT operation_module, COUNT(*) as count 
FROM operation_log 
GROUP BY operation_module 
ORDER BY count DESC;
```

## ⚠️ 注意事项

1. **日志表维护**
   - 建议定期清理历史日志（如保留90天）
   - 监控表大小，必要时进行归档

2. **性能考虑**
   - 不要在高频接口上记录响应结果
   - 避免记录过大的参数

3. **安全性**
   - 操作日志查询接口仅限管理员访问
   - 删除操作需要谨慎使用
