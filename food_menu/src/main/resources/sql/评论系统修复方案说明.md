# 评论系统修复方案说明

## 问题描述

1. **用户改名后，之前的评论还显示旧名字**：评论表中存储了 `author_name`，但用户改名后这些字段不会自动更新
2. **头像没有更新**：评论表中存储了 `avatar_url`，但用户改头像后这些字段不会自动更新
3. **被回复人改名后显示旧名字**：`reply_to_name` 只存储了名字，没有存储用户ID，导致被回复人改名后也会显示旧名字

## 解决方案

### 1. 数据库结构优化

- **添加 `reply_to_user_id` 字段**：存储被回复人的用户ID，用于实时获取最新用户信息
- **保留历史字段**：`author_name`、`avatar_url`、`reply_to_name` 作为历史快照保留，但查询时优先使用实时数据

### 2. 查询逻辑优化

- **实时获取用户信息**：查询评论时，根据 `wx_user_id` 和 `reply_to_user_id` 从 `wx_user` 表实时获取最新的用户名和头像
- **兼容历史数据**：如果用户不存在或查询失败，使用评论表中存储的历史数据作为兜底

### 3. 保存逻辑优化

- **自动设置被回复人ID**：保存回复评论时，根据 `parent_id` 自动查找被回复评论的作者ID，并设置到 `reply_to_user_id`
- **保存用户信息快照**：仍然保存用户名和头像作为历史记录，但查询时优先使用实时数据

## 执行步骤

### 步骤1：执行数据库迁移SQL

执行 `fix_comment_user_info.sql` 文件中的SQL语句：

```sql
-- 1. 添加 reply_to_user_id 字段
ALTER TABLE `dish_comment` 
ADD COLUMN `reply_to_user_id` bigint NULL DEFAULT NULL COMMENT '被回复人用户ID' AFTER `reply_to_name`;

-- 2. 根据现有数据填充 reply_to_user_id
UPDATE `dish_comment` dc1
INNER JOIN `dish_comment` dc2 ON dc1.parent_id = dc2.id
SET dc1.reply_to_user_id = dc2.wx_user_id
WHERE dc1.parent_id IS NOT NULL AND dc1.reply_to_user_id IS NULL;

-- 3. 添加索引（可选，提高查询性能）
CREATE INDEX `idx_reply_to_user_id` ON `dish_comment`(`reply_to_user_id`);
```

### 步骤2：重新编译后端代码

```bash
cd food_menu
mvn clean install -DskipTests
```

### 步骤3：重启后端服务

重启 Spring Boot 应用，使代码更改生效。

## 效果说明

修复后的效果：

1. **实时显示最新用户名**：用户改名后，所有历史评论都会显示新的用户名
2. **实时显示最新头像**：用户改头像后，所有历史评论都会显示新的头像
3. **被回复人信息实时更新**：被回复人改名后，所有回复都会显示新的用户名

## 注意事项

1. **性能考虑**：查询评论时会批量查询用户信息，使用 `IN` 查询，性能影响较小
2. **数据兼容**：完全兼容旧数据，如果用户不存在或查询失败，会使用历史数据作为兜底
3. **历史记录保留**：`author_name`、`avatar_url`、`reply_to_name` 字段仍然保留，作为历史快照
