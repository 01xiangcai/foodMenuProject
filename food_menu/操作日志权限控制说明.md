# 操作日志权限控制说明

## 概述

操作日志系统已实现基于角色的权限控制和数据隔离，确保不同角色的管理员只能查看和操作相应范围的操作日志。

## 角色定义

根据 `User` 实体类中的 `role` 字段：

| 角色 | role值 | 说明 |
|------|--------|------|
| 普通管理员 | 0 | 普通管理员 |
| 家庭管理员 | 1 | 家庭管理员 |
| 超级管理员 | 2 | 超级管理员 |

## 权限规则

### 超级管理员 (role=2)

**权限范围**：
- ✅ 可以查看**所有家庭**的操作日志
- ✅ 可以通过 `familyId` 参数筛选特定家庭的日志
- ✅ 可以查看、删除所有日志
- ✅ 可以批量删除所有家庭的日志

**使用场景**：
- 系统审计
- 跨家庭数据分析
- 全局日志管理

**示例**：
```bash
# 查看所有日志
GET /admin/operationLog/page?page=1&pageSize=10

# 查看特定家庭的日志
GET /admin/operationLog/page?familyId=1

# 删除任意日志
DELETE /admin/operationLog/123
```

### 普通管理员/家庭管理员 (role=0/1)

**权限范围**：
- ✅ 只能查看**自己家庭**的操作日志
- ❌ 无法查看其他家庭的日志
- ✅ 只能删除自己家庭的日志
- ❌ 无法通过 `familyId` 参数查看其他家庭的日志（会被自动过滤）

**使用场景**：
- 家庭内部操作审计
- 家庭数据管理
- 家庭操作追溯

**示例**：
```bash
# 查看自己家庭的日志（familyId参数会被忽略）
GET /admin/operationLog/page?page=1&pageSize=10
# 系统自动过滤：只返回当前用户家庭的日志

# 尝试查看其他家庭的日志（会被拒绝）
GET /admin/operationLog/page?familyId=999
# 系统自动过滤：只返回当前用户家庭的日志，忽略familyId=999
```

## 实现原理

### 1. Token解析

系统从请求头 `Authorization: Bearer {token}` 中获取Token，并解析：
- `role`：用户角色
- `familyId`：用户所属家庭ID

### 2. 权限检查

在 `OperationLogController` 中：

```java
// 获取用户角色和家庭ID
Integer userRole = jwtUtil.getRole(token);
Long userFamilyId = jwtUtil.getFamilyId(token);

// 判断是否为超级管理员
boolean isSuperAdmin = (userRole != null && userRole == 2);

// 数据过滤
if (!isSuperAdmin && userFamilyId != null) {
    // 非超级管理员：只能查看自己家庭的数据
    queryWrapper.eq(OperationLog::getFamilyId, userFamilyId);
} else if (isSuperAdmin) {
    // 超级管理员：可以按familyId筛选（如果提供了参数）
    if (queryDto.getFamilyId() != null) {
        queryWrapper.eq(OperationLog::getFamilyId, queryDto.getFamilyId());
    }
}
```

### 3. 数据隔离

- **查询接口**：在SQL查询时自动添加 `family_id` 条件
- **详情接口**：检查日志的 `family_id` 是否匹配
- **删除接口**：检查日志的 `family_id` 是否匹配

## 安全特性

### 1. 服务端验证

- ✅ 所有权限检查在**服务端**进行
- ✅ 前端无法绕过权限控制
- ✅ Token验证确保用户身份

### 2. 自动过滤

- ✅ 非超级管理员的查询请求自动添加家庭ID过滤
- ✅ 即使前端传递了其他家庭的ID，也会被自动忽略
- ✅ 确保数据隔离的可靠性

### 3. 错误处理

- ❌ Token无效：返回空结果或权限错误
- ❌ 无权限访问：返回 "无权查看该操作日志"
- ❌ 日志不存在：返回 "操作日志不存在"

## 前端集成建议

### 1. 请求头设置

```javascript
// 在所有操作日志相关请求中添加Token
axios.get('/admin/operationLog/page', {
  params: { page: 1, pageSize: 10 },
  headers: {
    'Authorization': `Bearer ${localStorage.getItem('token')}`
  }
})
```

### 2. 权限判断

```javascript
// 根据用户角色显示不同的功能
const userRole = getUserRole(); // 从Token或用户信息中获取

if (userRole === 2) {
  // 超级管理员：显示家庭筛选器
  showFamilyFilter();
} else {
  // 普通管理员：隐藏家庭筛选器
  hideFamilyFilter();
}
```

### 3. 错误处理

```javascript
// 处理权限错误
axios.get('/admin/operationLog/123')
  .then(res => {
    if (res.data.code === 0 && res.data.msg.includes('无权')) {
      // 显示权限不足提示
      showPermissionDeniedMessage();
    }
  })
  .catch(err => {
    // 处理其他错误
    handleError(err);
  });
```

## 注意事项

1. **Token必须有效**
   - 所有接口都需要有效的Token
   - Token过期会导致权限检查失败

2. **家庭ID必须存在**
   - 普通管理员和家庭管理员必须有 `familyId`
   - 如果 `familyId` 为null，将无法查看任何日志

3. **日志必须关联家庭**
   - 操作日志的 `family_id` 字段必须正确设置
   - 如果日志没有 `family_id`，非超级管理员将无法查看

4. **数据一致性**
   - 确保操作日志记录时正确设置了 `family_id`
   - 建议在 `OperationLogAspect` 中自动设置
