# 家宴菜单系统知识库

## 一、系统概述

**家宴菜单**是一套面向多家庭的餐饮管理平台，分为两个端：

- **管理后台（Admin）**：供管理员在 PC 端进行菜品管理、订单处理、用户管理、营销活动运营等。
- **微信小程序（UniApp）**：供家庭成员点餐、查看菜单、参与活动、管理个人钱包等。

---

## 二、多家庭模式

系统以**家庭（Family）为隔离单元**，所有数据（菜品、订单、用户、营销活动）均归属于某个家庭，家庭之间数据完全隔离，互不可见。

- 每个用户（小程序端）必须加入某个家庭后才能正常使用点餐功能。
- 每个家庭可以独立配置菜单、餐次规则、营销活动。

---

## 三、用户角色说明

### 小程序用户（微信用户）

| 角色 | 说明 |
|------|------|
| 普通成员（role=0） | 可点餐、查看菜单、参与活动、使用钱包 |
| 家庭管理员（role=1） | 拥有家庭内部的管理权限 |

### 管理后台用户（Admin）

| 角色 | 权限范围 |
|------|----------|
| 超级管理员 | 可查看所有家庭数据，可按家庭分类查看 |
| 家庭管理员 | 只能管理自己所属家庭的数据，权限高于普通管理员 |
| 普通管理员 | 只能管理自己所属家庭的数据 |

---

## 四、菜品管理

### 菜品属性

| 字段 | 说明 |
|------|------|
| 名称 | 菜品名称 |
| 分类 | 所属分类（如：主食、荤菜、素菜等） |
| 价格 | 单价 |
| 图片 | 支持多张图片（本地存储） |
| 描述 | 菜品详细介绍 |
| 卡路里 | 营养信息 |
| 标签 | 自定义标签（如：招牌、辣、低脂） |
| 状态 | 起售（1）/ 停售（0） |
| 排序权重 | 控制显示顺序 |

### 菜品分类

管理员可自定义菜品分类，用于在小程序端按分类展示菜单。

### 菜品标签

标签按家庭维度管理，可对菜品贴多个标签，小程序端可按标签筛选菜品。

---

## 五、口味/规格（DishFlavor）

每道菜品可以配置多种口味或规格（如辣度、大小份），用户点餐时可按需选择。

## 六、订单管理

### 订单状态流转

```
待支付(5) → 待接单(0) → 准备中(1) → 配送中(2) → 已完成(3)
                                              ↓
                                          已取消(4)
```

| 状态码 | 含义 |
|--------|------|
| 0 | 待接单 |
| 1 | 准备中 |
| 2 | 配送中 |
| 3 | 已完成 |
| 4 | 已取消 |
| 5 | 待支付 |

### 支付方式

| 方式 | 说明 |
|------|------|
| 余额支付（1） | 使用钱包余额扣款 |
| 模拟支付（2） | 测试/演示用途 |

### 支付状态

| 状态码 | 含义 |
|--------|------|
| 0 | 未支付 |
| 1 | 已支付 |
| 2 | 已退款 |

### 迟到订单

若用户在截止时间后仍提交订单，系统标记为**迟到订单**，需家庭管理员审核：

| 迟到订单状态 | 含义 |
|-------------|------|
| 0 | 待审核 |
| 1 | 已接受 |
| 2 | 已拒绝 |

## 七、每日点餐（大订单）

大订单（DailyMealOrder）是一种**按家庭、按日期、按餐次**汇总的集合订单，管理员发布当日菜单后，家庭成员在规定时间内点餐，系统自动汇总。

### 餐次类型

| 餐次 | 说明 |
|------|------|
| BREAKFAST | 早餐 |
| LUNCH | 午餐 |
| DINNER | 晚餐 |

### 大订单状态

| 状态码 | 含义 |
|--------|------|
| 0 | 收集中（成员可点餐） |
| 1 | 已确认 |
| 2 | 已截止（超过截止时间） |
| 3 | 已出餐 |

### 餐次时间配置

每个家庭可独立配置三餐的**开始时间、结束时间、下单截止时间**，超过截止时间后自动关闭当餐点餐入口（迟到订单需审核）。

---

## 八、用户钱包

每位小程序用户拥有独立钱包，支持以下功能：

| 功能 | 说明 |
|------|------|
| 余额查询 | 查看当前可用余额和冻结金额 |
| 充值 | 管理员可为用户充值 |
| 支付密码 | 使用余额支付时需验证支付密码（BCrypt加密存储） |
| 冻结金额 | 下单后金额冻结，确认完成后扣除 |
| 交易流水 | 可查看充值、消费、退款记录 |

**安全机制**：支付密码每日错误次数有上限，超限后当日锁定，次日自动重置。

## 九、营销活动

系统支持多种营销活动类型，每个家庭可独立运营：

| 活动类型 | 说明 |
|----------|------|
| LOTTERY | 抽奖活动 |
| WHEEL | 大转盘 |
| COUPON | 优惠券发放 |
| POINTS_EXCHANGE | 积分兑换 |
| SIGN_IN | 签到活动 |
| GROUP_BUY | 拼团活动 |

### 活动状态

| 状态码 | 含义 |
|--------|------|
| 0 | 未开始 |
| 1 | 进行中 |
| 2 | 已结束 |
| 3 | 已暂停 |

### 参与限制

- 支持每日限制次数（DAILY）、总次数限制（TOTAL）、不限次数（UNLIMITED）三种模式。
- 可配置参与条件（如最低消费、指定商品购买等）。

- **收藏（Favorite）**：用户可收藏喜欢的菜品，方便下次快速点餐。

## 十四、AI 助手功能

系统内置 AI 智能助手（基于 Qwen/Qwen2.5-7B-Instruct 大模型）：

- **菜单 AI 推荐**：根据用户口味偏好、历史订单智能推荐菜品。
- **AI 点餐对话**：用户可通过自然语言描述需求，AI 自动匹配菜品。
- **聊天记录**：AI 对话历史可查询。

## 十八、常见问题（FAQ）

### Q1：我的订单一直显示"待接单"，什么情况？
**A：** 待接单状态表示您的订单已提交成功，正在等待家庭管理员或厨房接单。通常很快会有状态更新，如长时间未变更可联系管理员。

### Q2：我的余额支付失败了是什么原因？
**A：** 可能原因：① 钱包余额不足；② 支付密码错误次数超限（当日锁定，次日重置）；③ 订单金额大于可用余额（冻结金额不可用）。

### Q3：我提交了订单但超过了截止时间，订单还有效吗？
**A：** 超过截止时间提交的订单会被标记为"迟到订单"，需要家庭管理员审核后才能进入正常流程。如被拒绝，订单将取消并退款。

### Q4：如何加入一个家庭？
**A：** 请联系您家庭的管理员，由管理员在后台将您的账户添加到对应家庭中。

### Q5：我的菜品图片无法显示？
**A：** 系统图片使用本地存储，请联系管理员确认后端服务是否正常运行，以及图片存储路径配置是否正确。

### Q6：如何修改支付密码？
**A：** 进入小程序 → 个人中心 → 钱包 → 安全设置 → 修改支付密码。

### Q7：忘记支付密码怎么办？
**A：** 请联系家庭管理员，由管理员在后台为您重置支付密码。

### Q8：营销活动参与次数用完了还能参与吗？
**A：** 如限制类型为"每日"，次日自动重置可继续参与；如为"总计"限制，则活动次数用完后无法再参与，需等待下期活动。

### Q9：我收藏的菜品在哪里查看？
**A：** 小程序端首页 → 我的收藏，可查看所有已收藏的菜品，并可直接加入购物车。

### Q10：可以为菜品选择口味吗？
**A：** 可以。进入菜品详情页，如果该菜品配置了口味/规格选项（如辣度、份量），可在点餐时进行选择。

### Q11：订单已完成但金额有误，如何申请退款？
**A：** 请联系家庭管理员，由管理员在后台处理退款操作。退款金额将原路返回到您的钱包余额。

### Q12：我是管理员，如何设置每日点餐截止时间？
**A：** 登录管理后台 → 餐次配置 → 选择对应餐次（早/中/晚）→ 设置开始时间、结束时间和下单截止时间。
