# 标签管理系统修复说明

## 问题描述

管理后台的标签管理页面出现以下错误：
```
Uncaught SyntaxError: The requested module '/src/api/http.ts' does not provide an export named 'http'
Uncaught TypeError: Cannot read properties of null (reading 'setAttribute')
```

## 问题原因

1. **API 导入方式错误**：`TagsView.vue` 使用了命名导入 `import { http } from '@/api/http'`，但 `http.ts` 使用的是默认导出 `export default http`
2. **API 调用方式不统一**：直接使用 `http` 实例而不是使用 `modules.ts` 中定义的 API 函数
3. **API 路径不一致**：前端使用 `/dishTag`，后端使用 `/dish-tag`

## 修复内容

### 1. 更新 API Modules (`food_menu_admin/src/api/modules.ts`)

添加了标签管理相关的 TypeScript 类型定义和 API 函数：

```typescript
export type DishTagQuery = {
  page: number;
  pageSize: number;
  name?: string;
};

export type DishTagPayload = {
  id?: number;
  name: string;
  icon: string;
  sort?: number;
};

export const fetchDishTags = (params: DishTagQuery) => http.get('/dish-tag/page', { params });
export const fetchAllDishTags = () => http.get('/dish-tag/list');
export const createDishTag = (data: DishTagPayload) => http.post('/dish-tag', data);
export const updateDishTag = (data: DishTagPayload) => http.put('/dish-tag', data);
export const removeDishTag = (id: number) => http.delete(`/dish-tag/${id}`);
```

### 2. 重写 TagsView 组件 (`food_menu_admin/src/views/tags/TagsView.vue`)

**主要改进：**

1. **正确的 API 导入**：
```typescript
import { fetchDishTags, createDishTag, updateDishTag, removeDishTag, type DishTagPayload } from '@/api/modules';
```

2. **简化的数据结构**：
```typescript
interface Tag {
  id?: number;
  name: string;
  icon: string;
  sort: number;
  createTime?: string;
}
```

3. **使用 API 函数而不是直接调用 http**：
```typescript
// 加载标签
const res = await fetchDishTags(params);

// 创建标签
await createDishTag(payload);

// 更新标签
await updateDishTag(payload);

// 删除标签
await removeDishTag(tag.id);
```

4. **优化的 UI 设计**：
   - 现代化的卡片式布局
   - 响应式表格设计
   - 优雅的对话框动画
   - Emoji 图标选择器
   - 分页控制

### 3. 后端 API 路径统一

确保所有 API 调用使用 `/dish-tag` 路径，与后端 Controller 的 `@RequestMapping("/dish-tag")` 保持一致。

## 后端 API 接口说明

### 管理端接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/dish-tag/page` | 分页查询标签列表 |
| GET | `/dish-tag/list` | 获取所有启用的标签 |
| GET | `/dish-tag/{id}` | 获取标签详情 |
| POST | `/dish-tag` | 新增标签 |
| PUT | `/dish-tag` | 更新标签 |
| DELETE | `/dish-tag/{id}` | 删除标签 |
| PUT | `/dish-tag/status/{id}` | 更新标签状态 |

### 小程序端接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/dish-tag/list` | 获取启用的标签列表 |
| GET | `/dish-tag/icon-map` | 获取标签图标映射 |

## 数据库表结构

```sql
CREATE TABLE `dish_tag` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键',
  `name` varchar(50) NOT NULL COMMENT '标签名称',
  `icon` varchar(50) NULL DEFAULT NULL COMMENT '标签图标(Emoji)',
  `type` int NULL DEFAULT 5 COMMENT '标签类型: 1-菜系, 2-口味, 3-特点, 4-分类, 5-其他',
  `sort` int NOT NULL DEFAULT 0 COMMENT '排序',
  `status` int NULL DEFAULT 1 COMMENT '状态: 0-禁用, 1-启用',
  `create_time` datetime NULL DEFAULT NULL COMMENT '创建时间',
  `update_time` datetime NULL DEFAULT NULL COMMENT '更新时间',
  `create_user` bigint NULL DEFAULT NULL COMMENT '创建人',
  `update_user` bigint NULL DEFAULT NULL COMMENT '修改人',
  PRIMARY KEY (`id`) USING BTREE,
  UNIQUE INDEX `uk_name`(`name` ASC) USING BTREE,
  INDEX `idx_type`(`type` ASC) USING BTREE,
  INDEX `idx_status`(`status` ASC) USING BTREE
) ENGINE = InnoDB CHARACTER SET = utf8mb4 COLLATE = utf8mb4_0900_ai_ci COMMENT = '菜品标签配置表';
```

## 使用说明

### 管理员操作流程

1. **访问标签管理页面**
   - 在管理后台侧边栏点击"标签管理"

2. **添加新标签**
   - 点击"新增标签"按钮
   - 输入标签名称
   - 选择或输入 Emoji 图标
   - 设置排序号（可选）
   - 点击"保存"

3. **编辑标签**
   - 在标签列表中点击"编辑"按钮
   - 修改标签信息
   - 点击"保存"

4. **删除标签**
   - 在标签列表中点击"删除"按钮
   - 确认删除操作

### 小程序端效果

标签图标会自动显示在：
- 菜品列表的菜品卡片上
- 菜品详情页的信息区域

图标由管理员在后台配置，前端自动获取并显示。

## 注意事项

1. **标签名称唯一性**：数据库中标签名称有唯一索引，不能重复
2. **图标格式**：建议使用 Emoji 字符，也可以使用图片 URL
3. **排序规则**：数字越小越靠前
4. **状态控制**：禁用的标签不会在小程序端显示
5. **类型分类**：标签分为菜系、口味、特点、分类、其他五种类型
