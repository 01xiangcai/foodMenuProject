## 权限控制实现

### 超级管理员 (role=2)

- ✅ 显示家庭筛选器
- ✅ 可以查看所有家庭的日志
- ✅ 可以通过家庭筛选器筛选特定家庭
- ✅ 可以批量删除日志

### 普通管理员/家庭管理员 (role=0/1)

- ❌ 隐藏家庭筛选器
- ✅ 只能查看自己家庭的日志
- ✅ 无法查看其他家庭的日志
- ❌ 无法批量删除（按钮隐藏）

## 使用说明

### 1. 访问操作日志页面

1. 登录管理后台
2. 在侧边栏点击「操作日志」
3. 或直接访问 `/operationLogs`

### 2. 筛选日志

**超级管理员**：
- 可以选择家庭进行筛选
- 可以选择操作类型、操作模块、状态等
- 可以选择时间范围
- 可以输入操作人姓名搜索

**普通管理员**：
- 可以选择操作类型、操作模块、状态等
- 可以选择时间范围
- 可以输入操作人姓名搜索
- 家庭筛选器自动隐藏，系统自动过滤

### 3. 查看日志详情

1. 点击表格中的「详情」按钮
2. 在弹出的 Modal 中查看：
   - 操作基本信息
   - 请求参数（JSON格式）
   - 响应结果（JSON格式）
   - 错误信息（如果有）

### 4. 删除日志

**单个删除**：
1. 点击表格中的「删除」按钮
2. 确认删除

**批量删除**（仅超级管理员）：
1. 点击「批量删除」按钮
2. 选择截止时间
3. 确认删除

### 1. 组件库

使用 **Naive UI** 组件：
- `NDataTable` - 数据表格
- `NSelect` - 下拉选择
- `NDatePicker` - 日期选择

- 判断用户角色（`isSuperAdmin`）
- 根据角色显示/隐藏功能

### 3. API 调用

使用封装的 `http` 实例：
- 自动添加 `Authorization` 请求头
- 统一错误处理
- 返回数据格式统一

## 注意事项

1. **Token 必须有效**
   - 所有 API 请求都需要有效的 Token
   - Token 过期会导致请求失败

2. **权限检查**
   - 前端根据用户角色显示/隐藏功能
   - 后端会进行二次权限验证
   - 即使前端绕过，后端也会拒绝未授权请求

3. **数据隔离**
   - 普通管理员和家庭管理员无法查看其他家庭的日志
   - 即使修改前端代码，后端也会自动过滤

4. **批量删除**
   - 仅超级管理员可以使用
   - 删除操作不可恢复，请谨慎操作
