## 📝 使用说明

### 开发环境

开发环境可以直接使用，配置已经设置好，不需要额外操作。

### 生产环境部署

#### 步骤1：生成加密值

```bash
cd food_menu
mvn test -Dtest=JasyptEncryptorTest
```

复制输出的加密值（格式：`ENC(密文)`）

#### 步骤2：更新生产配置

编辑 `application-prod.yml`，将占位符替换为实际加密值：

```yaml
spring:
  datasource:
    password: ENC(这里替换为实际加密值)

jwt:
  secret: ENC(这里替换为实际加密值)

aliyun:
  oss:
    access-key-id: ENC(这里替换为实际加密值)
    access-key-secret: ENC(这里替换为实际加密值)
```

#### 步骤3：设置环境变量

启动应用前，设置加密密钥环境变量：

```bash
# Linux/Mac
export JASYPT_ENCRYPTOR_PASSWORD=FoodMenuSecretKey2024!@#

# Windows
set JASYPT_ENCRYPTOR_PASSWORD=FoodMenuSecretKey2024!@#
```

**重要：** 加密密钥要与生成加密值时使用的密钥一致！

#### 步骤4：启动应用

```bash
java -jar food_menu-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

## 🔧 常见问题

### Q1: 登录时提示"Key argument cannot be null"？

**原因：** JWT密钥未正确加载

**解决方案：**
1. 确认 `application.yml` 中配置了 `jwt.secret`
2. 确认启动类添加了 `@EnableConfigurationProperties(JwtProperties.class)`
3. 重启应用

### Q2: 启动时提示解密失败？

**原因：** 未设置环境变量或密钥不匹配

**解决：**
```bash
export JASYPT_ENCRYPTOR_PASSWORD=FoodMenuSecretKey2024!@#
```

### Q3: 如何修改加密密钥？

1. 修改 `JasyptEncryptorTest.java` 中的 `PASSWORD` 常量
2. 重新运行测试生成新的加密值
3. 更新配置文件
4. 更新环境变量

### Q4: 开发环境也需要设置环境变量吗？

不需要。开发环境配置了默认密钥 `DefaultPasswordChangeInProduction`，可以直接使用。

### Q5: Docker部署如何设置环境变量？

```bash
docker run -e JASYPT_ENCRYPTOR_PASSWORD=你的密钥 \
  -p 8080:8080 food-menu:latest
```

或在 `docker-compose.yml` 中：
```yaml
services:
  food-menu:
    environment:
      - JASYPT_ENCRYPTOR_PASSWORD=你的密钥
```

## 🐛 已修复的问题

### 问题1：启动时提示"Key argument cannot be null"

**错误信息：**
```
ERROR --- [food_menu] [nio-8080-exec-1] c.y.food_menu.controller.UserController  : 登录失败: Key argument cannot be null.
```

**原因：**
- `JwtProperties` 配置类未被正确加载
- Spring Boot 3 的 `@PostConstruct` 导入错误（使用了 javax 而非 jakarta）

**解决方案：**
1. 在 `FoodMenuApplication` 添加 `@EnableConfigurationProperties(JwtProperties.class)` 注解
2. 修改 `JwtUtil` 中的导入：`javax.annotation.PostConstruct` → `jakarta.annotation.PostConstruct`

**验证：**
- 应用启动成功
- 登录功能正常
- Token生成和验证正常
