# 菜品标签选择功能说明

## 功能概述

将菜品管理页面的"菜品标签"输入框改为多选下拉框，管理员可以直接从已配置的标签列表中选择，而不需要手动输入。

## 修改内容

### 1. UI 改进

**修改前：**
```vue
<NFormItem label="菜品标签">
  <NInput
    v-model:value="dishModal.form.tags"
    placeholder="用逗号分隔，如: 暖胃,低油,家常"
  />
</NFormItem>
```

**修改后：**
```vue
<NFormItem label="菜品标签">
  <NSelect
    v-model:value="dishModal.form.tagsArray"
    :options="tagOptions"
    multiple
    filterable
    placeholder="选择菜品标签"
    :loading="tagsLoading"
  />
</NFormItem>
```

### 2. 数据结构调整

#### 添加标签相关状态
```typescript
// 标签相关
const dishTags = ref<Array<{ id: number; name: string; icon: string }>>([]);
const tagsLoading = ref(false);
const tagOptions = computed(() => {
  return dishTags.value.map(tag => ({
    label: `${tag.icon} ${tag.name}`,  // 显示图标和名称
    value: tag.name                     // 值为标签名称
  }));
});
```

#### 表单数据结构
```typescript
const dishModal = reactive({
  // ... 其他字段
  form: {
    // ... 其他字段
    tags: '',                    // 保留原有字段，用于与后端交互
    tagsArray: [] as string[]    // 新增字段，用于多选组件
  }
});
```

### 3. API 集成

#### 导入标签 API
```typescript
import {
  // ... 其他导入
  fetchAllDishTags
} from '@/api/modules';
```

#### 加载标签列表
```typescript
// 加载标签列表
const loadDishTags = async () => {
  tagsLoading.value = true;
  try {
    const result = await fetchAllDishTags();
    dishTags.value = result.data || [];
  } catch (error) {
    console.error('加载标签失败:', error);
  } finally {
    tagsLoading.value = false;
  }
};
```

#### 页面初始化
```typescript
onMounted(async () => {
  await loadCategories();
  await loadDishes();
  await loadDishTags();  // 加载标签列表
});
```

### 4. 数据转换逻辑

#### 编辑时：字符串 → 数组
```typescript
const fillDishForm = (dish: any) => {
  // ... 其他字段赋值
  dishModal.form.tags = dish.tags || '';
  // 将标签字符串转换为数组
  dishModal.form.tagsArray = dish.tags 
    ? dish.tags.split(',').map((t: string) => t.trim()).filter(Boolean) 
    : [];
};
```

#### 保存时：数组 → 字符串
```typescript
const saveDish = async () => {
  // ... 验证逻辑
  
  // 将标签数组转换为字符串
  const tagsString = dishModal.form.tagsArray.length > 0 
    ? dishModal.form.tagsArray.join(',') 
    : null;
  
  const payload: any = {
    // ... 其他字段
    tags: tagsString
  };
  
  // ... 保存逻辑
};
```

#### 重置表单
```typescript
const resetDishForm = () => {
  // ... 其他字段重置
  dishModal.form.tags = '';
  dishModal.form.tagsArray = [];
  // ... 其他重置逻辑
};
```

## 功能特性

### 1. 多选功能
- ✅ 支持选择多个标签
- ✅ 已选标签显示为标签组
- ✅ 可以删除已选标签

### 2. 搜索过滤
- ✅ 支持输入关键词搜索标签
- ✅ 实时过滤匹配的标签

### 3. 图标显示
- ✅ 下拉选项显示标签图标和名称
- ✅ 例如：`🌶️ 川菜`、`🔥 辣`

### 4. 加载状态
- ✅ 加载标签时显示加载动画
- ✅ 防止重复加载

### 5. 数据兼容
- ✅ 与后端数据格式完全兼容（字符串格式）
- ✅ 前端使用数组便于操作
- ✅ 自动转换，无需修改后端

## 使用流程

### 管理员操作

1. **新增菜品**
   - 点击"新增菜品"按钮
   - 在"菜品标签"字段点击下拉框
   - 从列表中选择一个或多个标签
   - 可以使用搜索功能快速查找标签
   - 保存菜品

2. **编辑菜品**
   - 点击菜品的"编辑"按钮
   - 系统自动加载已选标签
   - 可以添加或删除标签
   - 保存修改

3. **查看标签**
   - 下拉框显示所有已配置的标签
   - 每个标签显示图标和名称
   - 已选标签在输入框中显示

## 技术细节

### 数据流转

```
后端数据 (字符串)
    ↓
"川菜,辣,下饭"
    ↓
前端解析 (fillDishForm)
    ↓
["川菜", "辣", "下饭"]
    ↓
用户编辑 (NSelect 多选)
    ↓
["川菜", "辣", "下饭", "家常"]
    ↓
前端转换 (saveDish)
    ↓
"川菜,辣,下饭,家常"
    ↓
后端保存 (字符串)
```

### 标签选项格式

```typescript
tagOptions = [
  { label: "🌶️ 川菜", value: "川菜" },
  { label: "🔥 辣", value: "辣" },
  { label: "🍚 下饭", value: "下饭" },
  // ...
]
```

### 组件属性说明

```vue
<NSelect
  v-model:value="dishModal.form.tagsArray"  <!-- 绑定标签数组 -->
  :options="tagOptions"                      <!-- 标签选项列表 -->
  multiple                                   <!-- 启用多选 -->
  filterable                                 <!-- 启用搜索过滤 -->
  placeholder="选择菜品标签"                  <!-- 占位提示 -->
  :loading="tagsLoading"                     <!-- 加载状态 -->
/>
```

## 优势对比

### 修改前（手动输入）
- ❌ 需要记住标签名称
- ❌ 容易输入错误
- ❌ 标签不统一（如"辣"和"辣味"）
- ❌ 没有图标提示
- ❌ 需要手动输入逗号分隔

### 修改后（下拉选择）
- ✅ 直观的下拉选择
- ✅ 标签名称统一规范
- ✅ 图标辅助识别
- ✅ 支持搜索快速查找
- ✅ 多选操作便捷
- ✅ 自动处理分隔符

## 注意事项

1. **标签来源**
   - 标签必须先在"标签管理"页面配置
   - 只显示已启用的标签

2. **数据格式**
   - 后端存储格式：`"标签1,标签2,标签3"`（逗号分隔字符串）
   - 前端操作格式：`["标签1", "标签2", "标签3"]`（数组）

3. **兼容性**
   - 完全兼容现有数据
   - 不影响已有菜品的标签显示
   - 可以与手动输入的标签共存

4. **性能优化**
   - 标签列表在页面加载时一次性获取
   - 使用 computed 计算标签选项
   - 避免重复请求
