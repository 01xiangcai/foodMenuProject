# 家宴菜单系统部署文档

## 一、系统要求

- **操作系统**: CentOS 7.5+ / Ubuntu 20.04+ / Debian 10+
- **Java**: OpenJDK 17
- **数据库**: MySQL 8.0+ (或 5.7+)
- **Web服务器**: Nginx
- **Node.js**: 18.0+ (用于前端构建)
- **推荐配置**: 2核CPU / 4GB内存 / 40GB硬盘

> 💡 **本文档已针对 CentOS 7.5 x86_64 系统优化**,所有命令和配置均已适配。

### 2.3 宝塔面板部署详细步骤

#### 步骤1: 安装宝塔面板

**CentOS 7.5 系统安装宝塔面板:**

```bash
# 安装宝塔面板 (CentOS 7.x)
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh ed8484bec
```

安装完成后,会显示面板地址、用户名和密码,请妥善保存。

访问面板地址并登录(例如: `http://服务器IP:8888/xxxxxxxx`)

#### 步骤2: 安装运行环境

在宝塔面板中安装以下软件:

1. **软件商店** → 搜索并安装:
   - **MySQL 8.0** (或 5.7) - 选择"极速安装"
   - **Nginx 1.22** (或更高版本) - 选择"极速安装"
   - **Java项目管理器** (如果有)
   - **Supervisor 管理器** (推荐,用于进程守护)

2. **安装 Java 17** (CentOS 7.5):

   在宝塔终端中执行:

   ```bash
   # 方法一: 使用yum安装OpenJDK 17 (推荐)
   # 先添加EPEL仓库
   yum install -y epel-release
   
   # 安装Java 17
   yum install -y java-17-openjdk java-17-openjdk-devel
   
   # 验证安装
   java -version
   
   # 设置JAVA_HOME环境变量
   echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk' >> /etc/profile
   echo 'export PATH=$JAVA_HOME/bin:$PATH' >> /etc/profile
   source /etc/profile
   ```

   如果yum仓库没有Java 17,使用方法二:

   ```bash
   # 方法二: 手动下载安装
   cd /usr/local
   wget https://download.oracle.com/java/17/latest/jdk-17_linux-x64_bin.tar.gz
   tar -zxvf jdk-17_linux-x64_bin.tar.gz
   mv jdk-17* java17
   
   # 配置环境变量
   echo 'export JAVA_HOME=/usr/local/java17' >> /etc/profile
   echo 'export PATH=$JAVA_HOME/bin:$PATH' >> /etc/profile
   source /etc/profile
   
   # 验证
   java -version
   ```

#### 步骤3: 创建数据库

1. 进入 **数据库** 菜单
2. 点击 **添加数据库**
3. 填写信息:
   - 数据库名: `food_menu`
   - 用户名: `food_menu_user`
   - 密码: 设置强密码(记录下来,后续配置需要)
   - 访问权限: `本地服务器`
   - 字符集: `utf8mb4`
4. 点击 **提交**

#### 步骤4: 导入数据库表结构

1. 点击数据库后面的 **管理** 按钮
2. 进入 **phpMyAdmin**
3. 选择 `food_menu` 数据库
4. 点击 **导入** 选项卡
5. 选择项目的 SQL 文件并导入

或使用命令行:

```bash
# 在宝塔终端执行
mysql -u food_menu_user -p food_menu < /path/to/schema.sql
```

#### 步骤5: 创建网站和目录

1. **创建网站**:
   - 进入 **网站** 菜单
   - 点击 **添加站点**
   - 域名: 填写你的域名或服务器IP
   - 根目录: `/www/wwwroot/food-menu-admin`
   - PHP版本: 纯静态
   - 点击 **提交**

2. **创建后端目录**:

   在宝塔终端或文件管理器中执行:

   ```bash
   # 创建目录
   mkdir -p /www/wwwroot/food-menu-backend
   mkdir -p /www/wwwroot/food-menu-backend/uploads
   mkdir -p /www/wwwroot/food-menu-backend/uploads/images
   mkdir -p /www/wwwroot/food-menu-backend/uploads/avatars
   
   # 设置权限 (CentOS使用www用户)
   chown -R www:www /www/wwwroot/food-menu-backend/uploads
   chmod -R 755 /www/wwwroot/food-menu-backend/uploads
   ```

#### 步骤6: 上传并配置后端

1. **编译后端项目** (在本地):

   ```bash
   cd food_menu
   mvn clean package -DskipTests
   ```

2. **上传JAR包**:
   - 使用宝塔 **文件** 管理器
   - 上传 `target/food_menu-0.0.1-SNAPSHOT.jar` 到 `/www/wwwroot/food-menu-backend/`
   - 重命名为 `food_menu.jar`

3. **修改配置文件**:
   - 编辑 `food_menu/src/main/resources/application-prod.yml`
   - 修改数据库连接信息:

   ```yaml
   spring:
     datasource:
       url: jdbc:mysql://localhost:3306/food_menu?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
       username: food_menu_user
       password: ENC(加密后的密码)  # 使用步骤3创建的数据库密码加密后的值
   
   file:
     upload:
       path: /www/wwwroot/food-menu-backend/uploads
       image-path: /www/wwwroot/food-menu-backend/uploads/images
       avatar-path: /www/wwwroot/food-menu-backend/uploads/avatars
   ```

4. **创建启动脚本**:

   在 `/www/wwwroot/food-menu-backend/` 创建 `start.sh`:

   ```bash
   #!/bin/bash
   export JASYPT_ENCRYPTOR_PASSWORD=你的加密密钥
   nohup java -Xms512m -Xmx1024m -jar /www/wwwroot/food-menu-backend/food_menu.jar \
     --spring.profiles.active=prod \
     > /www/wwwroot/food-menu-backend/app.log 2>&1 &
   echo $! > /www/wwwroot/food-menu-backend/app.pid
   ```

   设置执行权限:

   ```bash
   chmod +x /www/wwwroot/food-menu-backend/start.sh
   ```

5. **创建停止脚本**:

   创建 `stop.sh`:

   ```bash
   #!/bin/bash
   if [ -f /www/wwwroot/food-menu-backend/app.pid ]; then
       kill $(cat /www/wwwroot/food-menu-backend/app.pid)
       rm -f /www/wwwroot/food-menu-backend/app.pid
       echo "应用已停止"
   else
       echo "应用未运行"
   fi
   ```

   设置执行权限:

   ```bash
   chmod +x /www/wwwroot/food-menu-backend/stop.sh
   ```

#### 步骤7: 使用宝塔进程守护

##### 方法一: 使用宝塔自带的进程守护

1. 进入 **软件商店** → 安装 **进程守护管理器**
2. 添加守护进程:
   - 名称: `food-menu`
   - 启动文件: `/www/wwwroot/food-menu-backend/start.sh`
   - 运行目录: `/www/wwwroot/food-menu-backend`
3. 点击 **启动**

##### 方法二: 使用 Supervisor (推荐)

1. 安装 Supervisor:
   - **软件商店** → 搜索 **Supervisor** → 安装

2. 添加守护进程:
   - 进入 Supervisor 管理界面
   - 点击 **添加守护进程**
   - 填写配置:

   ```ini
   [program:food-menu]
   directory=/www/wwwroot/food-menu-backend
   command=/usr/bin/java -Xms512m -Xmx1024m -jar /www/wwwroot/food-menu-backend/food_menu.jar --spring.profiles.active=prod
   environment=JASYPT_ENCRYPTOR_PASSWORD="你的加密密钥"
   autostart=true
   autorestart=true
   user=www
   stdout_logfile=/www/wwwroot/food-menu-backend/supervisor.log
   stderr_logfile=/www/wwwroot/food-menu-backend/supervisor_error.log
   ```

3. 点击 **保存** 并 **启动**

#### 步骤8: 部署前端

1. **本地构建前端**:

   ```bash
   cd food_menu_admin
   npm install
   npm run build
   ```

2. **上传前端文件**:
   - 使用宝塔 **文件** 管理器
   - 将 `dist` 目录下的所有文件上传到 `/www/wwwroot/food-menu-admin/`

#### 步骤9: 配置 Nginx 反向代理

1. 进入 **网站** 菜单
2. 找到刚创建的站点,点击 **设置**
3. 进入 **配置文件** 选项卡
4. 在 `server` 块中添加以下配置:

   ```nginx
   # API 反向代理
   location /api/ {
       proxy_pass http://127.0.0.1:8080/;
       proxy_set_header Host $host;
       proxy_set_header X-Real-IP $remote_addr;
       proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
       proxy_set_header X-Forwarded-Proto $scheme;
       
       proxy_connect_timeout 60s;
       proxy_send_timeout 60s;
       proxy_read_timeout 60s;
       
       client_max_body_size 20M;
   }
   
   # 上传文件静态访问
   location /uploads/ {
       alias /www/wwwroot/food-menu-backend/uploads/;
       expires 30d;
       add_header Cache-Control "public, immutable";
   }
   ```

5. 点击 **保存**

#### 步骤10: 配置 SSL 证书(可选)

1. 在网站设置中,进入 **SSL** 选项卡
2. 选择以下方式之一:
   - **Let's Encrypt**: 免费证书,自动续期
   - **其他证书**: 上传已有证书
   - **宝塔SSL**: 购买宝塔证书
3. 部署证书后,开启 **强制HTTPS**

#### 步骤11: 配置防火墙

**在宝塔面板中配置** (推荐):

1. 进入 **安全** 菜单
2. 确保以下端口已开放:
   - `80` (HTTP)
   - `443` (HTTPS)
   - `8888` (宝塔面板,可修改)
3. **不要**开放 `8080` 端口(后端端口,仅内网访问)

**或使用命令行配置** (CentOS 7.5 使用 firewalld):

```bash
# 启动firewalld
systemctl start firewalld
systemctl enable firewalld

# 开放端口
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --permanent --add-port=8888/tcp

# 重载防火墙
firewall-cmd --reload

# 查看已开放端口
firewall-cmd --list-ports
```

#### 步骤12: 设置定时备份

1. **数据库备份**:
   - 进入 **数据库** 菜单
   - 点击数据库后的 **备份** 按钮
   - 或在 **计划任务** 中添加数据库备份任务

2. **文件备份**:
   - 进入 **计划任务**
   - 添加 **备份目录** 任务
   - 选择备份 `/www/wwwroot/food-menu-backend/uploads`

#### 步骤13: 验证部署

1. **检查后端服务**:

   ```bash

   ps aux | grep food_menu
   
   # 查看日志
   tail -f /www/wwwroot/food-menu-backend/app.log

   ```

2. **访问前端**:
   - 浏览器访问: `http://你的域名` 或 `http://服务器IP`

3. **测试文件上传**:
   - 登录管理后台
   - 尝试上传图片
   - 检查 `/www/wwwroot/food-menu-backend/uploads/` 目录

#### 宝塔部署常见问题

**Q1: Java 进程启动后自动停止?**

检查内存是否足够:

```bash
# 查看内存使用
free -h

# 如果内存不足,减小JVM内存配置
# 修改启动脚本中的 -Xms512m -Xmx1024m 为更小的值
```

**Q2: Nginx 502 Bad Gateway?**

```bash
# 检查后端是否运行
ps aux | grep food_menu

# 检查端口是否监听
netstat -tlnp | grep 8080

# 查看后端日志
tail -f /www/wwwroot/food-menu-backend/app.log
```

**Q3: 文件上传失败?**

```bash
# 检查目录权限
ls -la /www/wwwroot/food-menu-backend/uploads

# 修复权限
chown -R www:www /www/wwwroot/food-menu-backend/uploads
chmod -R 755 /www/wwwroot/food-menu-backend/uploads
```

**Q4: 如何查看应用日志?**

在宝塔面板中:

- **Supervisor方式**: Supervisor管理界面 → 点击进程名 → 查看日志
- **手动启动方式**: 文件管理器 → 查看 `/www/wwwroot/food-menu-backend/app.log`

**Q5: 如何重启应用?**

- **Supervisor方式**: Supervisor管理界面 → 点击重启按钮
- **手动方式**:

  ```bash
  cd /www/wwwroot/food-menu-backend
  ./stop.sh
  ./start.sh
  ```

**Q6: CentOS系统SELinux导致权限问题?**

```bash
# 检查SELinux状态
getenforce

# 临时关闭SELinux
setenforce 0

# 永久关闭SELinux (需要重启)
sed -i 's/SELINUX=enforcing/SELINUX=disabled/g' /etc/selinux/config

# 或者配置SELinux策略允许访问
chcon -R -t httpd_sys_rw_content_t /www/wwwroot/food-menu-backend/uploads
```

**Q7: CentOS防火墙阻止访问?**

```bash
# 检查防火墙状态
systemctl status firewalld

# 查看已开放端口
firewall-cmd --list-all

# 如果端口未开放,参考步骤11配置防火墙
```

#### 宝塔部署运维技巧

1. **监控面板**: 宝塔面板首页可查看CPU、内存、磁盘使用情况
2. **日志分析**: 使用宝塔的日志分析工具查看访问日志和错误日志
3. **性能优化**: 在 **软件商店** → **Nginx** → **性能调整** 中优化Nginx配置
4. **定期更新**: 定期更新宝塔面板和已安装的软件
5. **安全加固**:
   - 修改宝塔面板默认端口
   - 启用面板SSL
   - 定期更新面板密码
   - 配置IP白名单
6. **CentOS特别注意**:
   - 关闭或正确配置SELinux,避免权限问题
   - 使用firewalld管理防火墙,不要使用iptables
   - 定期执行 `yum update` 更新系统补丁
   - 检查系统日志: `journalctl -xe` 或 `/var/log/messages`

### 2.4 手动部署

#### 步骤1: 安装依赖

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Java 17
sudo apt install -y openjdk-17-jdk

# 安装 MySQL
sudo apt install -y mysql-server

# 安装 Nginx
sudo apt install -y nginx

# 安装 Node.js (前端构建用)
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
```

#### 步骤2: 配置数据库

```bash
# 登录 MySQL
sudo mysql

# 创建数据库和用户
CREATE DATABASE food_menu DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'food_menu_user'@'localhost' IDENTIFIED BY '你的密码';
GRANT ALL PRIVILEGES ON food_menu.* TO 'food_menu_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

# 导入数据库表结构(如果有SQL文件)
mysql -u food_menu_user -p food_menu < schema.sql
```

#### 步骤3: 部署后端

```bash
# 创建部署目录
sudo mkdir -p /opt/food-menu
sudo mkdir -p /opt/food-menu/uploads  # 文件上传目录
sudo mkdir -p /opt/food-menu/uploads/images  # 图片存储
sudo mkdir -p /opt/food-menu/uploads/avatars  # 头像存储
cd /opt/food-menu

# 设置目录权限
sudo chown -R www-data:www-data /opt/food-menu/uploads
sudo chmod -R 755 /opt/food-menu/uploads

# 编译后端项目(在本地或服务器上)
cd food_menu


# 上传 JAR 包到服务器
scp target/food_menu-0.0.1-SNAPSHOT.jar user@server:/opt/food-menu/food_menu.jar

# 修改生产配置文件
# 编辑 food_menu/src/main/resources/application-prod.yml
# 配置数据库连接、文件存储路径等信息

# 创建 systemd 服务
sudo nano /emvn clean package -DskipTeststc/systemd/system/food-menu.service
```

**food-menu.service 内容:**

```ini
[Unit]
Description=Food Menu Backend Service
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/food-menu
ExecStart=/usr/bin/java -Xms512m -Xmx1024m -jar /opt/food-menu/food_menu.jar --spring.profiles.active=prod
Restart=on-failure
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

```bash
# 启动后端服务
sudo systemctl daemon-reload
sudo systemctl start food-menu
sudo systemctl enable food-menu

# 查看服务状态
sudo systemctl status food-menu

# 查看日志
sudo journalctl -u food-menu -f
```

#### 步骤4: 部署管理后台前端

```bash
# 在本地构建前端
cd food_menu_admin
npm install
npm run build

# 上传构建产物到服务器
scp -r dist/* user@server:/www/wwwroot/food-menu-admin/

# 或在服务器上创建目录
sudo mkdir -p /www/wwwroot/food-menu-admin
sudo chown -R www-data:www-data /www/wwwroot/food-menu-admin
```

#### 步骤5: 配置 Nginx

```bash
# 复制配置文件
sudo cp nginx-food-menu.conf /etc/nginx/sites-available/food-menu

# 修改配置中的域名/IP
sudo nano /etc/nginx/sites-available/food-menu
```

**关键配置 - 添加静态文件访问:**

```nginx
# 上传文件静态访问
location /uploads/ {
    alias /opt/food-menu/uploads/;
    expires 30d;
    add_header Cache-Control "public, immutable";
}
```

```bash
# 启用站点
sudo ln -s /etc/nginx/sites-available/food-menu /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
sudo systemctl enable nginx
```

#### 步骤6: 配置防火墙

```bash
sudo ufw allow 22/tcp   # SSH
sudo ufw allow 80/tcp   # HTTP
sudo ufw allow 443/tcp  # HTTPS
sudo ufw enable
```

#### 步骤7: 配置 HTTPS (可选)

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 申请证书(替换为你的域名)
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo systemctl enable certbot.timer
```

## 三、生产环境配置修改指南

### 3.1 必须修改的配置参数

在部署到生产环境之前,**必须**修改以下配置参数:

#### 配置文件位置

`food_menu/src/main/resources/application-prod.yml`

#### 必改参数清单

| 参数 | 说明 | 示例值 | 是否加密 |
|------|------|--------|---------|
| `spring.datasource.url` | 数据库连接地址 | `jdbc:mysql://localhost:3306/food_menu...` | 否 |
| `spring.datasource.username` | 数据库用户名 | `food_menu_user` | 否 |
| `spring.datasource.password` | 数据库密码 | `ENC(加密后的密码)` | **是** |
| `jwt.secret` | JWT签名密钥 | `ENC(加密后的JWT密钥)` | **是** |
| `file.upload.path` | 文件上传路径 | `/opt/food-menu/uploads` | 否 |
| `logging.file.path` | 日志文件路径 | `/var/log/food_menu` | 否 |

#### 可选参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `server.port` | 服务端口 | `8080` |
| `spring.datasource.hikari.maximum-pool-size` | 数据库连接池大小 | `20` |
| `jwt.expiration-time` | JWT过期时间(毫秒) | `604800000` (7天) |

### 3.2 配置加密说明

生产环境的敏感信息(数据库密码、JWT密钥等)必须使用 **Jasypt** 加密。

#### 步骤1: 生成加密值

```bash
# 进入后端项目目录
cd food_menu

# 运行加密测试类
mvn test -Dtest=JasyptEncryptorTest
```

**或在IDE中直接运行** `src/test/java/.../JasyptEncryptorTest.java`

**输出示例:**

```text
=== Jasypt 加密测试 ===
加密密钥: FoodMenuSecretKey2024!@#

明文: root
密文: xMjQwMDAwMDAw...
配置文件格式: ENC(xMjQwMDAwMDAw...)

明文: YourJWTSecretKey
密文: yNjUwMDAwMDAw...
配置文件格式: ENC(yNjUwMDAwMDAw...)
```

#### 步骤2: 修改配置文件

将生成的加密值替换到 `application-prod.yml`:

```yaml
spring:
  datasource:
    password: ENC(xMjQwMDAwMDAw...)  # 替换为实际加密后的数据库密码

jwt:
  secret: ENC(yNjUwMDAwMDAw...)  # 替换为实际加密后的JWT密钥
```

#### 步骤3: 设置解密密钥

生产环境启动时,必须提供解密密钥。有以下三种方式:

#### 方式一: 环境变量(推荐)

```bash
# 设置环境变量
export JASYPT_ENCRYPTOR_PASSWORD=FoodMenuSecretKey2024!@#

# 启动应用
java -jar food_menu.jar --spring.profiles.active=prod
```

#### 方式二: Systemd服务配置(推荐用于生产)

编辑 `/etc/systemd/system/food-menu.service`:

```ini
[Service]
Environment="JASYPT_ENCRYPTOR_PASSWORD=FoodMenuSecretKey2024!@#"
ExecStart=/usr/bin/java -jar /opt/food-menu/food_menu.jar --spring.profiles.active=prod
```

#### 方式三: 启动参数

```bash
java -jar -Djasypt.encryptor.password=FoodMenuSecretKey2024!@# \
  food_menu.jar --spring.profiles.active=prod
```

> ⚠️ **重要提示:**
>
> - 加密密钥(`JASYPT_ENCRYPTOR_PASSWORD`)必须与生成加密值时使用的密钥一致
> - 加密密钥不要提交到Git仓库
> - 建议使用16位以上复杂密码,包含大小写字母、数字、特殊字符

```bash
sudo chown -R www-data:www-data /opt/food-menu/uploads

sudo nginx -t
```

## 十、安全建议

1. **修改默认密码**: 数据库、管理员账号等
