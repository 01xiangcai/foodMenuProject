# 标签管理系统实现说明

## 📋 概述

将标签图标配置从前端硬编码改为后台管理，实现了完整的标签管理系统。管理员可以在后台自由添加、编辑、删除标签，并为每个标签配置专属的 Emoji 图标。

## 🏗️ 系统架构

### 1. 数据库层

#### 表结构：`dish_tag`
```sql
CREATE TABLE `dish_tag` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '主键',
  `name` varchar(50) NOT NULL COMMENT '标签名称',
  `icon` varchar(50) NULL COMMENT '标签图标(Emoji)',
  `type` int DEFAULT 5 COMMENT '标签类型: 1-菜系, 2-口味, 3-特点, 4-分类, 5-其他',
  `sort` int DEFAULT 0 COMMENT '排序',
  `status` int DEFAULT 1 COMMENT '状态: 0-禁用, 1-启用',
  `create_time` datetime NULL COMMENT '创建时间',
  `update_time` datetime NULL COMMENT '更新时间',
  `create_user` bigint NULL COMMENT '创建人',
  `update_user` bigint NULL COMMENT '修改人',
  PRIMARY KEY (`id`),
  UNIQUE INDEX `uk_name`(`name`),
  INDEX `idx_type`(`type`),
  INDEX `idx_status`(`status`)
);
```

#### 初始数据
- 预置 33 个常用标签
- 包含 5 种类型：菜系、口味、特点、分类、其他
- 每个标签都配置了对应的 Emoji 图标


#### Service 层
- `DishTagService.java` - 接口定义
- `DishTagServiceImpl.java` - 实现类

**核心方法：**
- `getEnabledTags()` - 获取所有启用的标签
- `getTagIconMap()` - 获取标签图标映射（name → icon）

**管理端接口：**
- `GET /dish-tag/page` - 分页查询标签列表

- `PUT /dish-tag` - 更新标签
- `DELETE /dish-tag/{id}` - 删除标签
- `PUT /dish-tag/status/{id}` - 更新标签状态

**小程序端接口：**
- `GET /dish-tag/list` - 获取启用标签列表

### 4. 小程序端

#### API 接口：`src/api/index.js`
```javascript
// 获取标签图标映射
export const getTagIconMap = () => {
  return request({
    url: '/dish-tag/icon-map',
    method: 'GET'
  })
}

// 获取标签列表
export const getTagList = (type) => {
  return request({
    url: '/dish-tag/list',
    method: 'GET',
    data: type ? { type } : {}
  })
}
```

#### 菜单页面：`menu.vue`
```javascript
// 加载标签图标映射
const tagIconMap = ref({})

const loadTagIconMap = async () => {
  try {
    const res = await getTagIconMap()
    if (res.data) {
      tagIconMap.value = res.data
    }
  } catch (error) {
    console.error('加载标签图标映射失败:', error)
    tagIconMap.value = {}
  }
}

// 获取标签图标
const getTagIcon = (tag) => {
  return tagIconMap.value[tag] || '🔸'
}
```

#### 详情页面：`detail.vue`
同样的实现方式，从后端获取标签图标映射。

## 🔄 工作流程

### 1. 管理员配置标签
1. 登录管理后台
2. 进入"标签管理"页面
3. 添加/编辑标签，选择 Emoji 图标
4. 设置标签类型、排序、状态
5. 保存配置

### 2. 小程序端使用标签
1. 页面加载时调用 `loadTagIconMap()`
2. 从后端获取标签图标映射
3. 缓存到 `tagIconMap` 变量
4. 渲染标签时通过 `getTagIcon(tag)` 获取图标
5. 如果标签未配置图标，显示默认图标 🔸

### 3. 数据流向
```
管理后台 → 后端数据库 → 后端 API → 小程序端 → 用户界面
   ↓           ↓            ↓          ↓          ↓
 配置标签    存储数据    提供接口   获取配置   显示图标
```

## 🎯 优势对比

### 优化前（硬编码）
❌ 图标固定在前端代码中  
❌ 修改需要重新发布  
❌ 无法动态管理  
❌ 扩展性差  
❌ 维护困难  

### 优化后（后台管理）
✅ 图标配置在数据库  
✅ 实时生效，无需发布  
✅ 动态增删改查  
✅ 扩展性强  
✅ 易于维护  
✅ 管理员自主配置  

## 🚀 使用指南

### 管理员操作

#### 1. 添加新标签
1. 点击"添加标签"按钮
2. 输入标签名称（如：香辣）
3. 选择或输入 Emoji 图标（如：🌶️）
4. 选择标签类型（如：口味）
5. 设置排序值（越小越靠前）
6. 选择状态（启用/禁用）
7. 点击"确定"保存

#### 2. 编辑标签
1. 找到要编辑的标签
2. 点击"✏️"编辑按钮
3. 修改标签信息
4. 点击"确定"保存

#### 3. 删除标签
1. 找到要删除的标签
2. 点击"🗑️"删除按钮
3. 确认删除操作

#### 4. 启用/禁用标签
1. 找到要操作的标签
2. 点击"🔓/🔒"按钮
3. 标签状态立即切换

## 📝 注意事项

### 1. 数据一致性
- 标签名称必须唯一
- 删除标签前确认没有菜品在使用
- 禁用标签不会影响已有菜品

### 2. 性能优化
- 小程序端缓存标签图标映射
- 避免频繁请求接口
- 页面加载时一次性获取
